python数据化运营

Posted on 2019-06-24

2.1 数据化运营的数据来源类型

  • 数据文件
  • 数据库
  • API
  • 流式数据
  • 外部公开数据
  • 其他来源数据

2.1.1 数据文件

数据文件就是存储数据的文件，广义数据是指任何文件储存的村西，狭义的数据是指以数字或文本形式储存的结构化数据记录。

2.1.2 数据库

数据库是按照数据结构来组织、存储和管理数据的仓库。

数据库的主要应用：

  • 事务性工作：定义、存储、增加、删除、更新、查询
  • 数据清洗工作： 数据传输、同步、抽取、转换、加载
  • 分析型工作： 数据计算、关联查询、OLAP
  • 其他工作： 数据权限控制、数据质量维护、异构数据库、多系统通信交互

数据库的分类：

  • 关系型数据库：DB2、Sybase、Oracle、SQL Server、MYSAQL
  • 非关系型数据库：
    • 面向高性能并发读写的键值数据库（Redis）
    • 面向海量文档的文档数据库（MongoDB）
    • 面向可扩展性的列式数据库（HBase）
    • 面向图结构的图形数据库（Neo4J）

2.1.3 API

API（Application Programming Interface）：应用程序编程接口
数据化运营中的API通常分为服务型API和数据型API

  • 服务型API：基于预定义的规则，通过调用API实现特定功能
  • 数据型API： 通过特定的语法，通过向服务器发送数据请求，返回特定格式的数据（大多数情况是JSON、XML）

JSON

  • 定义：一种轻量级的数据交换格式，由流行的JavaScript编程语言创建，广泛应用于Web数据交互。
  • 特点：格式简介、结构清晰、使用键值对的格式存储数据对象，键是数据对象的属性，值是数据对象属性的对应值。例如，“性别”：“男”。

XML：

  • 定义：可扩展标记语言，提供了统一的方法来描述和交换独立与应用程序或供应商的结构化数据，是一种成熟且强大的数据格式。
  • 特点： 有非常好的扩展性，API创建者可以使用他们创建自己的数据结构。

2.1.4 流式数据

流式数据

  • 定义：是指实时或接近实时处理的大数据流。常见的框架有Spark，Storm等。
  • 特点：能在毫秒到秒之间完成作业，用于处理时效性较强的场景。
  • 分类：
    • 用户行为数据流：围绕人产生的数据流，包括用户在网站和App内部因浏览、搜索、评论、分享、交易以及在外部的微博微信中操作而产生的数据流。
    • 机器数据流：围绕物产生的数据流，包括从机器的生产、制造、应用、监控和管理等过程中产生的所有数据。

2.1.5 外部公开数据

  • 定义：公开的任意第三方都能获取的数据
  • 分类：
    • 政府和相关机构提供的公开数据
    • 竞争对手主动公开的数据
    • 行业协会或相关平台组织提供的统计、咨询数据
    • 第三方的组织或个人披露的与企业运营相关的数据

2.1.6 其他

其他数据例如通过调研问卷获得的有关产品、客户等方面的数据，从第三方平台直接购买的数据，通过与其他厂商合作所得交互数据等，这些数据获取场景较少，而且不是主流的数据获取来源。

2.2 使用python获取运营数据

主要途径：

  • 文本文件
  • Excel文件
  • 关系型和非关系型数据库
  • API
  • 网页

2.2.1 从文本文件读取运营数据

  1. 使用read、readline、readlines读取数据
    基本步骤：
    1）定义数据文件
    2）获取文件对象
    3）读取文件内容
    4）关闭文件对象

  2. 使用Numpy的loadtext、load、fromfile读取数据

  3. 使用Pandas的read_csv、read_fwf、read_table读取数据

  4. 如何选择最佳读取数据的方法

判断依据：

  • 数据源情况
  • 数据处理目标
  • 模型数据要求
  • 手感最好的方法

2.2.2 从Excel获取运营数据

由于Excel本身的限制和适用范围，其无法储存和计算过大的数据量，并且Excel本身也不是为了海量数据的应用而产生的。因此Excel只适用于日常基本的数据处理
